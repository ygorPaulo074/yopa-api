Getting Started

API Reference

The AI-ChatBot API is a multi-tenant REST API for deploying and operating conversational agents. Each agent has its own configuration, session history, analytics, and optional knowledge base.

Base URL: http://your-host:8000

Authentication

All endpoints (except POST /agent) require an agent-specific API key passed as a request header.

Header: X-API-Key: <your-api-key>
The API key is generated once on agent creation. Store it securely — it cannot be recovered.

Quickstart

Create an agent, configure its context, and send the first message in three steps.

1. Create an agent

Request
POST /agent
Content-Type: application/json

{
  "name": "My Agent",
  "owner": "you@company.com"
}
Response
{
  "agent_id": "agt_...",
  "api_key": "sk_live_..."
}

2. Set the agent context

Request
PUT /agent/context
X-API-Key: sk_live_...

{
  "persona": "You are a helpful support agent for Loja X.",
  "tone": "formal",
  "language": "pt-BR"
}

3. Send a message

Request
POST /chat
X-API-Key: sk_live_...

{
  "session_id": "sess_abc123",
  "message": "What is your return policy?"
}
Response
{
  "message_id": "msg_...",
  "content": "Our return policy allows returns within 30 days...",
  "usage": { "input_tokens": 120, "output_tokens": 42, "total_tokens": 162 },
  "response_time_ms": 834
}
Agents

Agent Endpoints

POST /agent Create a new agent

Creates an agent and returns the API key. The key is shown only once.

FieldTypeDescription
namestringDisplay name for the agent
ownerstringOwner identifier (email or ID)
tagslist[string]Optional tags for filtering
PUT /agent/context Update agent context

Each update creates a new versioned snapshot. See the Agent context guide for all fields.

GET /agent/context Get current context

Returns the latest version of the agent context.

GET /agent/context/history List context versions

Returns all versioned snapshots of the context with change history.

GET /agent/metrics Agent metrics

Returns aggregated metrics: session count, token usage, average sentiment, escalation rate.

DELETE /agent Delete agent and all data

Permanently removes the agent, all sessions, contexts, and user data. Irreversible.

Chat

Chat Endpoints

POST /chat Send a message

Processes a user message and returns the AI response. Triggers NLP analysis and incremental persistence on every call.

FieldTypeDescription
session_idstringUnique session identifier (generate client-side)
user_idstring?Optional user identifier for context tracking
messagestringThe user's message text
POST /chat/{session_id}/end End a session

Finalizes the session, generates insights, and persists full history. Updates user context if user_id was provided.

POST /chat/{session_id}/escalate Escalate session

Marks the session as escalated. Can be triggered manually or automatically via escalation triggers.

POST /chat/{session_id}/resolve Resolve session

Marks the session as resolved without ending it.

Data

Data Endpoints

GET /data/chat List sessions

Returns all sessions for the authenticated agent.

GET /data/chat/{session_id} Get session history

Returns the full message history. Falls back to the persistence driver if the session has expired from Redis cache.

GET /data/chat/{session_id}/insights Session insights

Returns AI-generated insights for an ended session: key points, suggested actions, summary.

GET /data/analytics Analytics

Aggregated agent analytics: sessions over time, average sentiment, top topics, escalation rate.

Guides

Escalation Triggers

Escalation conditions are evaluated by the backend after every message — they are never sent to the AI model.

TypeParameterDescription
sentimentthreshold: floatEscalates if avg sentiment < -threshold
keywordvalues: list[str]Escalates if the last message contains any keyword
message_countvalue: intEscalates after N user messages
topicvalues: list[str]Escalates if detected topic is in the list
time_elapsedvalue: float (seconds)Escalates after N seconds since session start
intentvalue: strEscalates if detected intent matches
Example
{
  "operator": "OR",
  "conditions": [
    { "type": "sentiment", "threshold": 0.4 },
    { "type": "keyword", "values": ["refund", "cancel"] },
    { "type": "message_count", "value": 10 }
  ]
}
